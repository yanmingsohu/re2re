将以下 x86 汇编转换为可编译、可执行的 C 代码，严格遵守以下规则：

【调用约定 & 语法】
- 所有函数声明使用 __cdecl 调用约定。
- 保留的内联汇编块使用 MASM 语法（__asm { }）。
- 输出代码须能直接通过 MSVC 或 MinGW 编译，不留伪代码占位符。

【命名】
- 用有意义的名字替换 sub_XXXXXX 形式的函数名和 var_X 形式的变量名。
- 寄存器别名在首次赋值处替换为具名变量，行尾注释标注原寄存器。

【类型重建】
- 凡 int* 或 void* 存在固定偏移访问，须定义对应 struct 并全局替换。
- 字段类型与访问宽度对齐（uint32_t / uint16_t / uint8_t）。
- 若同一指针以不兼容布局访问，使用 union，并在阻断问题中说明。

【阻断规则 — 遇到下列情况立即停止，输出"⚠ 停止："并提问，禁止猜测继续】
1. 跳转目标超出已提供的 ASM 范围，或间接跳转目标未知。
2. 同一地址以不兼容宽度访问（类型冲突）。
3. 调用外部符号但未提供函数签名。
4. 栈指针偏移不平衡，局部变量布局无法确定。
5. 存在自修改代码或运行时解密桩。
