### 详细分析

#### 1. C 伪代码实现
根据汇编逻辑，该函数可以还原为如下易读的 C 代码：

<code>
void ClearDWORDArray(int* pBuffer, int elementCount)
{
    for (int i = 0; i < elementCount; ++i)
    {
        pBuffer[i] = 0;
    }
}
</code>

#### 2. 参数与寄存器映射表
通过分析栈偏移（`ESP+4`, `ESP+8`），可以确定该函数的调用约定为基于栈的参数传递（如 `__cdecl`）：

| 符号/地址 | 对应变量 | 类型 (猜测) | 说明 |
| :--- | :--- | :--- | :--- |
| `[ESP+4]` | `pBuffer` (EAX) | `int*` 或 `void*` | 指向待清零内存块起始地址的指针 |
| `[ESP+8]` | `elementCount` (ECX) | `int` / `DWORD` | 需要清零的 **元素个数**（注意：不是字节数） |
| `EAX` | 迭代指针 | `uint32_t*` | 在循环中不断自增，指向当前处理的内存地址 |
| `ECX` | 剩余计数器 | `int` | 记录还需要清零多少个元素 |

#### 3. 执行流程说明
1.  **初始化阶段**：从栈中取出第二个参数（元素个数）存入 `ECX`，取出第一个参数（目标地址）存入 `EAX`。
2.  **循环体 (`$L_502c48`)**：
    *   将 `EAX` 指向的内存地址处的值写入 `0` (`mov DWORD PTR [EAX], 0`)。
    *   将指针 `EAX` 向后移动 4 个字节，指向下一个元素 (`add EAX, 4`)。
    *   将计数器 `ECX` 减 1 (`dec ECX`)。
3.  **终止条件判断**：使用 `test ECX, ECX` 检查计数器是否为 0。如果 `ECX > 0`（即 `jg` 指令），则跳转回循环起始位置继续执行；否则退出循环并返回。
