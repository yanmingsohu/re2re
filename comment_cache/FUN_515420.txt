#### <code class="language-text">关键字段与指令解释</code>

| 指令/偏移 | 含义 | 技术细节说明 |
| :--- | :--- | :--- |
| `wait` | **FPU 同步** | 该指令会让 CPU 等待 FPU 完成当前正在进行的运算。在读取状态寄存器前调用它是为了防止“竞态条件”，确保读到的是最新的计算结果。 |
| `fnstsw` | **存储状态字** | `Store FPU Status Word` 的缩写。它将 FPU 状态寄存器的低 16 位拷贝到指定的内存地址中。 |
| `[EBP-2]` | **局部变量位置** | 指向栈上用于临时存放状态字的 16 位空间。注意这里使用了 `WORD PTR`，说明只取了低 16 位。 |
| `movsx EAX, ...` | **符号扩展** | 将获取到的 16 位 `WORD` 转换为 32 位的 `DWORD` 并存入 `EAX`。这意味着如果状态字最高位为 1，返回的整数将是负数。 |

1. **环境准备**：通过标准的函数序言（Prologue）建立栈帧，并保存了 `ECX` 的值（可能是为了遵循某种特定的调用约定或保护寄存器状态）。
2. **同步等待**：执行 `wait` 指令。这是典型的底层数学库写法，确保 FPU 处于空闲且计算已完成的状态。
3. **硬件读取**：利用 `fnstsw` 直接从 CPU 的浮点单元中抓取当前的状态位信息。
4. **结果转换**：将抓取到的原始 16 位数据进行符号扩展处理，放入返回值寄存器 `EAX` 中。
5. **清理返回**：通过 `leave` 指令快速恢复栈帧并销毁局部变量空间，随后执行 `ret` 返回。
