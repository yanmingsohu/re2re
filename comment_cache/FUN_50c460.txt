#### 2. 关键指令流程说明
该函数的精妙之处在于利用了 x86 的算术标志位（Carry Flag）来实现极其高效的布尔取反，而没有使用分支跳转指令（如 `jz` 或 `jnz`），这有助于避免 CPU 分支预测失败带来的性能损失。

| 汇编步骤 | 指令 | 操作逻辑描述 |
| :--- | :--- | :--- |
| **1. 取参** | `mov EAX, [ESP+4]` | 从栈中取出第一个参数（字符串指针）存入 `EAX`。 |
| **2. 调用** | `call FUN_50c410` | 调用相等性校验函数。结果以布尔值形式存在 `AL` 中（$1 = \text{True}, 0 = \text{False}$）。 |
| **3. 取反(算术)** | `neg AL` | 对 `AL` 执行求补码操作：<br>若 $AL=1 \rightarrow AL=0xFF$，且设置 $\text{CF}=1$<br>若 $AL=0 \rightarrow AL=0x00$，且设置 $\text{CF}=0$ |
| **4. 展开标志位** | `sbb EAX, EAX` | 使用减法并借位。计算公式为：$EAX = EAX - (EAX + CF)$，即 $EAX = -CF$<br>若 $\text{CF}=1 \rightarrow EAX=0xFFFFFFFF$<br>若 $\text{CF}=0 \rightarrow EAX=0x00000000$ |
| **5. 修正结果** | `inc EAX` | 将结果加 1：<br>若 $EAX=0xFFFFFFFF \rightarrow EAX=0$<br>若 $EAX=0x00000000 \rightarrow EAX=1$ |
| **6. 返回** | `ret 4` | 清理栈并返回结果。 |

#### 3. 数值演变追踪表
通过下表可以清晰看到逻辑是如何翻转的：

| 输入状态 (FUN_50c410) | 执行 `neg AL` 后 (AL, CF) | 执行 `sbb EAX, EAX` 后 (EAX) | 执行 `inc EAX` 后 (最终返回值) | 语义含义 |
| :--- | :--- | :--- | :--- | :--- |
| **1** (相等) | $0xFF, \text{CF}=1$ | $0xFFFFFFFF$ | **0** | 不相等 $\rightarrow$ 假 |
| **0** (不相等) | $0x00, \text{CF}=0$ | $0x00000000$ | **1** | 不相等 $\rightarrow$ 真 |
