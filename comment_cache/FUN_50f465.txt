void SetFPUControlWord(uint32_t val) {
    // 1. 保留输入参数中的特定位（第8和第9位，即 0x300）

#### 2. 关键字段与指令解释

| 指令 / 常量 | 含义说明 | 技术细节 |
| :--- | :--- | :--- |
| `and EDX, 768` | **位掩码过滤** | $768$ 即十六进制 `0x300`。这行代码的作用是**只保留输入参数中的第 8 位和第 9 位**，将其余所有位清零。 |
| `or EDX, 127` | **强制基础配置** | $127$ 即十六进制 `0x7F` (二进制 `0111 1111`)。这行代码确保了控制字中的**低 7 位被强制设为 1**。 |
| `fldcw` | **加载 FPU 控制字** | 这是核心指令。它将内存中的一个 `WORD`（16位）值加载到 x87 FPU 的控制寄存器中，直接改变后续浮点运算的行为（如精度、舍入模式）。 |
| `[ESP+4]` | **函数参数** | 在标准 `cdecl` 或 `stdcall` 调用约定下，这代表传入函数的第一个 32 位整数。 |

#### 3. 执行流程说明
1.  **提取输入**：从栈中获取调用者提供的原始配置值到 `EDX`。
2.  **位操作（过滤）**：通过 `AND 0x300` 剔除掉大部分用户自定义的设置，只留下对第 8、9 位的影响力。
3.  **位操作（填充）**：通过 `OR 0x7F` 强制注入一组预设的状态值到低位。
4.  **硬件应用**：将处理后的 16 位结果存入临时栈空间，并调用 `fldcw` 指令使该配置立即对 CPU 的浮点运算生效。
