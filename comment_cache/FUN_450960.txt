### 函数功能概括

**该函数是一个索引计算器，通过将两个输入参数（通常代表坐标或分块 ID）进行位运算和算术偏移，映射为一个 16 位的线性化数组索引。**

### 详细分析

#### 1. C 伪代码实现
根据汇编逻辑，该函数可以还原为如下 C 代码：

```c
unsigned short CalculateIndex(int raw_coord, int block_id) {
    // 1. 处理符号位对齐 (Sign-aware adjustment)
    // 如果 raw_coord 为负，则增加偏移量以修正 SAR 位移后的结果
    int sign_adj = (raw_coord < 0) ? 15 : 0;
    int adjusted_val = raw_coord + sign_adj;

    // 2. 计算子索引 (Sub-index)
    // 将坐标除以 16，并截断在 0-63 的范围内（即取低 6 位）
    unsigned int sub_index = (adjusted_val >> 4) & 0x3F;

    // 3. 计算主偏移量 (Main offset)
    // 取 block_id 的低 10 位，并左移 6 位（相当于乘以 64）
    unsigned int main_offset = (block_id & 0x3FF) << 6;

    // 4. 返回合并后的 16 位索引值
    return (unsigned short)(main_offset + sub_index);
}
```

#### 2. 关键逻辑拆解

| 操作步骤 | 汇编指令 | 逻辑含义 | 技术细节 |
| :--- | :--- | :--- | :--- |
| **符号修正** | `cdq`, `and EDX, 15`, `add EAX, EDX` | **负数对齐处理** | 通过 `cdq` 将符号扩展到 `EDX`，若原数为负则 `EDX=0xF`。这是一种实现类似于 `floor(x/16)` 的技巧，确保负坐标在位移后能正确映射回对应的格子。 |
| **子索引计算** | `sar EAX, 4`, `and EAX, 63` | **缩放与掩码** | 将输入值缩小 16 倍（右移 4 位），并限制其范围在 $0 \dots 63$ 之内。这表明每个“块”内部可能有 64 个单元。 |
| **主偏移计算** | `and ECX, 1023`, `shl ECX, 6` | **分块定位** | 将第二个参数限制在 $0 \dots 1023$（10位），然后乘以 64 ($2^6$)。这确定了当前索引属于哪一个大的数据分组。 |
| **合并返回** | `add ECX, EAX`, `mov AX, CX` | **线性化** | 将主偏移与子索引相加，得到最终的 $0 \dots 65535$ 的一维索引。 |
