### 2. 关键字段与寄存器解释

| 变量/寄存器 | 含义 | 说明 |
| :--- | :--- | :--- |
| `[EBP+12]` (ESI) | **输入流指针** (`pInput`) | 当前正在解析的字节流地址，函数执行后会更新此值。 |
| `[EBP-4]` | **状态标志位** (`statusFlags`) | 一个 Bitmask，记录了当前指令序列的状态（如：是否包含资源、是否已缩放等）。 |
| `[EBP+8]` | **流上下文指针** (`pStreamCtx`) | 传递给各种写入函数（`FUN_50e2d6`, `FUN_50e33c`）的目标输出对象。 |
| `BL` (ESI 的首字节) | **操作码** (`Opcode`) | 解析逻辑的“开关”，决定了后续所有的执行路径。 |
| `$L_517744` / `$L_517764` | **类型映射表** | 用于将原始 Opcode 转换为内部对象类型的查找数组。 |

### 3. 执行流程说明
1.  **提取 Header**: 从传入的指针处读取第一个字节，并立即递增指针以准备解析后续负载（Payload）。
2.  **逻辑分发 (The Big Switch)**:
    *   如果 Opcode 在特定范围内，通过查表获取一个索引，再根据该索引进行一次 `jmp DWORD PTR [EAX*4 + Offset]` 的间接跳转。这通常用于处理“类”或“对象类型”的实例化。
    *   如果是特定的 ASCII 字符（如 `'w'`, `'h'` 等），则通过位运算在本地标志变量中打桩，标记当前流的属性。
3.  **数据转换与输出**:
    *   解析过程中频繁调用 `FUN_50f5e0` (高精度缩放) 和 `FUN_512a60` (定点除法)。这表明该指令流包含大量需要精确计算的参数（例如：动画插值、坐标转换或颜色映射）。
    *   解析出的数据通过 `FUN_50e33c` (批量写入) 或 `FUN_50e2d6` (单字节写入) 转移到输出缓冲区中。
4.  **状态收尾**: 更新输入指针，使调用者知道当前流已消耗了多少字节。
