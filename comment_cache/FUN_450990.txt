### 函数功能概括
<b style="color: #e67e22;">该函数是一个高度优化的坐标/索引位包装函数 (Index/Coordinate Packing Function)，用于将四个离散的整数参数通过位移、掩码和符号偏置处理，合并成一个单一的 16 位无符号索引值。这种逻辑常见于游戏引擎中的纹理瓦片寻址（Tile Addressing）、Voxel（体素）空间索引或 2D/3D 网格坐标到线性内存地址的转换。</b>

#### 2. C 伪代码实现
通过对汇编逻辑的逐行还原，其数学逻辑如下：

<code style="display: block; background-color: #f8f9fa; padding: 15px; border-radius: 5px;">
unsigned short FUN_450990(int arg1, int arg2, int arg3, int arg4) {
    // --- 第一阶段：处理子偏移量 (Sub-offset Nibble) ---
    // 将 arg1 和 arg2 的低 2 位组合成一个 4 位的 nibble
    // 例如: arg1=0x3, arg2=0x1 -> nibble = 0b1101 (13)
    unsigned int nibble = ((arg1 & 3) << 2) | (arg2 & 3);

    // --- 第二阶段：处理参数 4 (主要分量 A) ---
    // 使用 cdq + and edx, 255 进行无分支符号补偿
    int sign_mask_d = (arg4 < 0) ? 0xFF : 0; 
    unsigned int part_d = (arg4 + sign_mask_d) >> 8;

    // 将 nibble 与 part_d 结合，nibble 被放大 2 倍后加到 part_d 上
    unsigned int intermediate = part_d + (nibble << 1);

    // --- 第三阶段：处理参数 3 (主要分量 B) ---
    int sign_mask_c = (arg3 < 0) ? 63 : 0;
    unsigned int part_c = (arg3 + sign_mask_c) >> 6;

    // --- 第四阶段：最终合并 ---
    // 将中间结果左移 4 位，并叠加分量 B 的值
    return (unsigned short)((intermediate << 4) | part_c);
}
</code>

#### 3. 流程逻辑说明
1.  **位打包 (Bit Packing)**: 函数首先利用 `arg1` 和 `arg2` 构建了一个极小的局部偏移（范围 0-15），这通常代表了在一个大块（Tile/Chunk）内部的微观坐标。
2.  **无分支符号处理 (Branchless Sign Handling)**: 代码中频繁使用 `cdq`（将 EAX 的符号位扩展到 EDX）配合 `and edx, mask` 之后再 `add eax, edx`。这是一种极其高效的技巧，避免了 CPU 分支预测失败带来的性能损失。它通过给负数加上一个特定的偏移量来将其“平移”到正数空间后再进行右移操作。
3.  **分层级缩放 (Hierarchical Scaling)**: 不同的参数使用了不同的位移步长（`>> 8`, `>> 6`, `<< 4`），这表明输入的分量具有不同的权重或精度，符合典型的 **Morton Order (Z-Order Curve)** 或类似的空间填充曲线的特征。
